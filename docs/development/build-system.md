# TurboRepo Build System Documentation

## Overview

CycleTime uses TurboRepo as its monorepo build orchestration system, providing parallel execution, intelligent caching, and dependency management across TypeScript packages. TurboRepo handles local development workflows, CI optimization, and ensures consistent build processes as the project scales from 2 to 8+ packages.

## Architecture

### Package Structure

**Current State:**
```
packages/
├── api-gateway/           # FastAPI gateway with auth
├── ai-service/            # AI provider abstraction with Worker Pool
├── shared-types/          # Common interfaces and type definitions
├── shared-utils/          # Utility functions and helpers
└── task-service/          # Linear and GitHub integration
```

**Dependency Hierarchy:**
1. **Shared Packages** (no internal dependencies)
   - `@cycletime/shared-types`
   - `@cycletime/shared-utils`

2. **Core Service Packages** (depend on shared packages)
   - `@cycletime/api-gateway`
   - `@cycletime/ai-service`

3. **Application Packages** (depend on core services + shared packages)
   - `@cycletime/task-service`
   - Future: `@cycletime/document-service`, `@cycletime/web-dashboard`

### TurboRepo Configuration

**Core Configuration (`turbo.json`):**
```json
{
  "$schema": "https://turbo.build/schema.json",
  "remoteCache": {
    "signature": true
  },
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", "coverage/**"],
      "cache": true
    },
    "test": {
      "dependsOn": ["build"],
      "outputs": ["coverage/**"],
      "cache": true
    },
    "test:unit": {
      "outputs": ["coverage/**"],
      "cache": true,
      "passThroughEnv": ["NODE_ENV"]
    },
    "test:integration": {
      "dependsOn": ["build"],
      "outputs": ["coverage/**"],
      "cache": true,
      "passThroughEnv": [
        "NODE_ENV", "DATABASE_URL", "REDIS_URL",
        "GITHUB_CLIENT_ID", "GITHUB_CLIENT_SECRET", 
        "GITHUB_REDIRECT_URI", "JWT_SECRET"
      ]
    },
    "lint": {
      "cache": true
    },
    "typecheck": {
      "dependsOn": ["^build"],
      "cache": true
    }
  },
  "cacheDir": ".turbo"
}
```

**Key Features:**
- **Dependency Graph**: `^build` ensures dependencies build before dependents
- **Parallel Execution**: Independent tasks run simultaneously
- **Output Caching**: Caches build artifacts (`dist/**`) and test coverage
- **Environment Passthrough**: Required environment variables for different test types
- **Remote Caching**: Signature verification enabled for security

## Local Development Workflows

### Core Commands

```bash
# Build all packages (respects dependency order)
turbo build

# Test all packages (parallel execution)
turbo test

# Lint all packages (parallel execution)
turbo lint

# Type check all packages
turbo typecheck

# Run all quality checks
turbo build lint typecheck test
```

### Package-Specific Operations

```bash
# Test specific package
turbo run test --filter=api-gateway

# Build specific package and its dependencies
turbo run build --filter=api-gateway...

# Test only unit tests across all packages
turbo run test:unit

# Test only integration tests across all packages
turbo run test:integration

# Multiple packages with pattern matching
turbo run test --filter="*-service"
```

### Affected Package Workflows

```bash
# Build only packages affected by recent changes
turbo run build --filter=...[HEAD~1]

# Test packages affected since main branch
turbo run test --filter=...[origin/main]

# Lint only changed packages
turbo run lint --filter=...[HEAD^]
```

### Development Server Management

```bash
# Start all development servers (parallel)
turbo run dev --parallel

# Start specific services
turbo run dev --filter=api-gateway --filter=web-dashboard --parallel

# Monitor logs from multiple services
turbo run dev --parallel --output-logs=stream
```

## Package Management

### Creating New Packages

**1. Directory Structure:**
```
packages/new-service/
├── package.json
├── tsconfig.json
├── jest.config.js
├── .eslintrc.js
├── src/
│   ├── index.ts
│   └── __tests__/

```

**2. Package.json Template:**
```json
{
  "name": "@cycletime/new-service",
  "version": "0.1.0",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "scripts": {
    "build": "tsc",
    "dev": "tsx watch src/index.ts",
    "test": "jest",
    "test:unit": "jest --testPathIgnorePatterns=integration",
    "test:integration": "jest --testPathPattern=integration",
    "lint": "eslint src/**/*.ts",
    "typecheck": "tsc --noEmit"
  },
  "dependencies": {
    "@cycletime/shared-types": "*",
    "@cycletime/shared-utils": "*"
  },
  "devDependencies": {
    "@types/node": "^20.6.0",
    "typescript": "^5.2.2",
    "jest": "^29.7.0",
    "eslint": "^8.49.0"
  }
}
```

**3. TypeScript Configuration:**
```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "references": [
    { "path": "../shared-types" },
    { "path": "../shared-utils" }
  ]
}
```

### Dependency Management Rules

**✅ Allowed Dependencies:**
- Application packages → Core services + Shared packages
- Core services → Shared packages only
- Shared packages → External dependencies only

**❌ Forbidden Dependencies:**
- Shared packages → Any internal packages
- Circular dependencies between any packages
- Direct cross-service dependencies (must go through shared packages)

## Caching Strategy

### Local Caching

**Cache Directory Structure:**
```
.turbo/
├── cache/              # Task output caches
├── daemon/             # TurboRepo daemon state
└── cookies/            # Remote cache authentication
```

**Cache Behavior:**
- **Build artifacts** cached in `dist/**`
- **Test coverage** cached in `coverage/**`
- **Cache keys** based on file hashes + dependency graph
- **Cache invalidation** on source file changes

### Remote Caching Setup

**1. Vercel Account Configuration:**
```bash
# Generate TurboRepo token from Vercel dashboard
# Add to environment variables:
export TURBO_TOKEN="your-vercel-token"
export TURBO_TEAM="your-team-slug"
```

**2. Authentication:**
```bash
# Login to Vercel (one-time setup)
npx turbo login

# Link project to remote cache
npx turbo link
```

**3. Verification:**
```bash
# First run - cache miss
turbo build
# Expected: "cache miss, executing abc123"

# Second run - cache hit
turbo build  
# Expected: "cache hit, replaying logs abc123"
```

### Cache Performance Optimization

**Cache Hit Strategies:**
- Use consistent Node.js versions across environments
- Ensure deterministic build outputs
- Exclude timestamps and environment-specific paths from outputs
- Use `.turboignore` for files that shouldn't affect cache keys

**Example `.turboignore`:**
```
# Logs and temporary files
*.log
.DS_Store
node_modules

# Environment-specific files
.env.local
.env.development
```

## Performance Optimization

### Build Performance

**Parallel Execution:**
- Independent packages build simultaneously
- Dependency graph ensures correct build order
- CPU cores automatically utilized

**Incremental Builds:**
- Only changed packages rebuilt
- Dependency propagation handled automatically
- Remote cache eliminates redundant work

**Performance Metrics:**
- **Cold build**: ~3-4 minutes for all packages
- **Incremental build**: ~30-60 seconds for affected packages
- **Cache hit**: ~5-10 seconds for replay
- **Parallel test execution**: 50-70% faster than sequential

### Memory Management

**Local Development:**
```bash
# Limit parallel processes for memory-constrained environments
turbo run test --concurrency=2

# Monitor memory usage
turbo run build --summarize

# Clear cache to free disk space
turbo prune
```

**CI Optimization:**
```bash
# Optimize for CI environment
turbo run build --cache-dir=.turbo --remote-only

# Skip local cache in CI (use remote only)
turbo run test --force
```

## Troubleshooting

### Common Issues

#### 1. Cache Misses Despite Unchanged Code

**Symptoms:** Always shows "cache miss" for unchanged packages

**Solutions:**
```bash
# Check cache configuration
npx turbo run build --dry-run

# Verify remote cache authentication
npx turbo login

# Check for non-deterministic outputs
turbo run build --summarize
```

#### 2. Build Dependency Issues

**Symptoms:** "Cannot find module" errors or incorrect build order

**Solutions:**
```bash
# Verify dependency graph
npx turbo run build --graph

# Check package.json dependencies
npm run build --workspace=@cycletime/package-name

# Clear and rebuild
turbo prune && turbo build
```

#### 3. Parallel Execution Problems

**Symptoms:** Tests failing in parallel but passing individually

**Solutions:**
```bash
# Run tests sequentially for debugging
turbo run test --concurrency=1

# Check for shared test resources
turbo run test --output-logs=full

# Isolate test environments
NODE_ENV=test turbo run test --filter=package-name
```

#### 4. Memory Issues

**Symptoms:** Out of memory errors during parallel execution

**Solutions:**
```bash
# Reduce concurrency
turbo run build --concurrency=2

# Increase Node.js memory
NODE_OPTIONS="--max-old-space-size=4096" turbo build

# Monitor memory usage
turbo run build --summarize
```

## Best Practices

### Development Workflow

1. **Always use TurboRepo commands** for multi-package operations
2. **Test affected packages** before committing: `turbo run test --filter=...[HEAD~1]`
3. **Use package filtering** for focused development: `turbo run dev --filter=api-gateway`
4. **Monitor cache performance** with `--summarize` flag periodically
5. **Keep shared packages minimal** to avoid circular dependencies

### Package Design

1. **Single responsibility** per package
2. **Clear API boundaries** through shared-types
3. **Consistent naming** for scripts across packages
4. **Minimal external dependencies** in shared packages
5. **Explicit dependency declarations** in package.json

### Performance Optimization

1. **Use affected package filtering** for PR workflows
2. **Enable remote caching** for all environments
3. **Optimize task dependencies** in turbo.json
4. **Monitor build performance** and adjust concurrency
5. **Regular cache cleanup** with `turbo prune`
